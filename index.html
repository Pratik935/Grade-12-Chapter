Chapter 1: Database Management System

Data

Data refers to raw facts, figures, or statistics that can be processed to produce meaningful information. Data can be in the form of numbers, text, images, audio, or video.

Information

Information is processed, organized, and structured data that provides context and meaning. It is useful for decision-making and understanding.

Features of Information

  • Accuracy: Information should be free from errors
  • Timeliness: Information should be available when needed
  • Relevance: Information should be pertinent to the user's needs
  • Completeness: Information should include all necessary details
  • Consistency: Information should be uniform across the system

Database and its Purpose

A database is an organized collection of related data that is stored and accessed electronically. The main purposes of a database are:

  • To store data efficiently and systematically
  • To provide easy access to data
  • To eliminate data redundancy
  • To maintain data integrity
  • To ensure data security

Terminologies in Database

  • Table: A collection of related data organized in rows and columns
  • Field: A single piece of information about a subject (column in a table)
  • Record: A complete set of fields about one subject (row in a table)
  • Tuple: Another term for a record in relational databases
  • Object: An instance of a class in object-oriented databases
  • Keys: Fields used to identify and establish relationships between tables

Data Dictionary

A data dictionary is a centralized repository of information about data such as meaning, relationships to other data, origin, usage, and format. It contains metadata (data about data) that helps in database management.

Database Management System (DBMS)

Introduction

A DBMS is software that interacts with users, applications, and the database itself to capture and analyze data. It provides an interface between the database and end users or application programs.

Objectives

  • To provide convenient and efficient environment for data access
  • To eliminate data redundancy and inconsistency
  • To facilitate data sharing among multiple users
  • To enforce data integrity and security
  • To provide backup and recovery facilities

Advantages

  • Reduced data redundancy
  • Improved data consistency
  • Better data security
  • Easier data sharing
  • Improved data integrity
  • Reduced application development time

Disadvantages

  • Complexity in design and implementation
  • High initial cost
  • Requires skilled personnel
  • Performance overhead
  • Vulnerability to failure

Types of Database Models

Hierarchical Model

Data is organized in a tree-like structure with parent-child relationships. Each parent can have many children, but each child has only one parent.

Network Model

An extension of the hierarchical model that allows many-to-many relationships. Data is represented as collections of records and relationships are represented by sets.

Relational Model

Data is organized in tables (relations) with rows and columns. Relationships between tables are established through foreign keys.

Entity-Relational Model

A conceptual model that describes data as entities, attributes, and relationships. It's often used as a design tool before implementing a relational database.

Integrity Constraints and Types

Integrity constraints are rules that maintain the accuracy and consistency of data in a database.

Domain Integrity

Ensures that all values in a column fall within defined valid values for that attribute (data type, format, range, etc.).

Entity Integrity

Ensures that each row in a table is uniquely identified (primary key cannot contain null values).

Referential Integrity

Ensures that relationships between tables remain consistent (foreign key must match a primary key in another table or be null).

Keys

  • Primary Key: Uniquely identifies each record in a table
  • Foreign Key: Establishes relationship between tables
  • Candidate Key: A field that could be a primary key
  • Alternate Key: A candidate key not chosen as primary key
  • Composite Key: A primary key made up of multiple columns

Normalization

Introduction

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. It involves dividing large tables into smaller, related tables and defining relationships between them.

Normal Forms

The stages of normalization are called normal forms:

1NF (First Normal Form)
  • Each table cell should contain a single value
  • Each record needs to be unique
  • No repeating groups or arrays
2NF (Second Normal Form)
  • Must be in 1NF
  • All non-key attributes must depend on the entire primary key (no partial dependency)
3NF (Third Normal Form)
  • Must be in 2NF
  • No transitive dependency (non-key attributes shouldn't depend on other non-key attributes)

Advantages

  • Reduced data redundancy
  • Improved data integrity
  • Better database organization
  • Easier database maintenance

Disadvantages

  • Increased complexity
  • More tables require more joins in queries
  • Potential performance impact

Centralized and Distributed Database

Introduction

Centralized Database: All data is stored at a single location and accessed by multiple users from that location.

Distributed Database: Data is stored across multiple locations (nodes) that are connected via network.

Advantages of Centralized Database

  • Easier to manage and maintain
  • Lower hardware and software costs
  • Better data consistency
  • Easier backup and recovery

Disadvantages of Centralized Database

  • Single point of failure
  • Performance bottleneck with many users
  • Network dependency for remote users

Advantages of Distributed Database

  • Improved performance (data closer to users)
  • Increased reliability (no single point of failure)
  • Scalability (easier to add more nodes)
  • Local autonomy (local control over local data)

Disadvantages of Distributed Database

  • More complex to design and implement
  • Higher hardware and software costs
  • More difficult to maintain data consistency
  • Security challenges

Database Security

Introduction

Database security refers to the protection of database from unauthorized access, misuse, or destruction. It includes various measures to ensure confidentiality, integrity, and availability of data.

Challenges

  • Unauthorized access
  • Data corruption
  • Data theft
  • SQL injection attacks
  • Denial of service attacks

Security Measures

  • Authentication: Verifying user identity
  • Authorization: Granting appropriate access rights
  • Encryption: Protecting sensitive data
  • Auditing: Tracking database activities
  • Backup and Recovery: Protecting against data loss

Roles of DBA (Database Administrator)

  • Installing and upgrading database software
  • Designing database structure
  • Implementing security measures
  • Managing user accounts and permissions
  • Monitoring database performance
  • Planning backup and recovery strategies
  • Troubleshooting database issues

Practical Topics: DDL and DML Languages

DDL (Data Definition Language): Used to define and modify database structure (CREATE, ALTER, DROP, TRUNCATE, RENAME).

DML (Data Manipulation Language): Used to manipulate data within database objects (SELECT, INSERT, UPDATE, DELETE).
