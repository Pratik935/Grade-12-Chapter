Chapter 1: Database Management System

Data

Data refers to raw facts, figures, or statistics that can be processed to produce meaningful information. Data can be in the form of numbers, text, images, audio, or video.

Information

Information is processed, organized, and structured data that provides context and meaning. It is useful for decision-making and understanding.

Features of Information

  • Accuracy: Information should be free from errors
  • Timeliness: Information should be available when needed
  • Relevance: Information should be pertinent to the user's needs
  • Completeness: Information should include all necessary details
  • Consistency: Information should be uniform across the system

Database and its Purpose

A database is an organized collection of related data that is stored and accessed electronically. The main purposes of a database are:

  • To store data efficiently and systematically
  • To provide easy access to data
  • To eliminate data redundancy
  • To maintain data integrity
  • To ensure data security

Terminologies in Database

  • Table: A collection of related data organized in rows and columns
  • Field: A single piece of information about a subject (column in a table)
  • Record: A complete set of fields about one subject (row in a table)
  • Tuple: Another term for a record in relational databases
  • Object: An instance of a class in object-oriented databases
  • Keys: Fields used to identify and establish relationships between tables

Data Dictionary

A data dictionary is a centralized repository of information about data such as meaning, relationships to other data, origin, usage, and format. It contains metadata (data about data) that helps in database management.

Database Management System (DBMS)

Introduction

A DBMS is software that interacts with users, applications, and the database itself to capture and analyze data. It provides an interface between the database and end users or application programs.

Objectives

  • To provide convenient and efficient environment for data access
  • To eliminate data redundancy and inconsistency
  • To facilitate data sharing among multiple users
  • To enforce data integrity and security
  • To provide backup and recovery facilities

Advantages

  • Reduced data redundancy
  • Improved data consistency
  • Better data security
  • Easier data sharing
  • Improved data integrity
  • Reduced application development time

Disadvantages

  • Complexity in design and implementation
  • High initial cost
  • Requires skilled personnel
  • Performance overhead
  • Vulnerability to failure

Types of Database Models

Hierarchical Model

Data is organized in a tree-like structure with parent-child relationships. Each parent can have many children, but each child has only one parent.

Network Model

An extension of the hierarchical model that allows many-to-many relationships. Data is represented as collections of records and relationships are represented by sets.

Relational Model

Data is organized in tables (relations) with rows and columns. Relationships between tables are established through foreign keys.

Entity-Relational Model

A conceptual model that describes data as entities, attributes, and relationships. It's often used as a design tool before implementing a relational database.

Integrity Constraints and Types

Integrity constraints are rules that maintain the accuracy and consistency of data in a database.

Domain Integrity

Ensures that all values in a column fall within defined valid values for that attribute (data type, format, range, etc.).

Entity Integrity

Ensures that each row in a table is uniquely identified (primary key cannot contain null values).

Referential Integrity

Ensures that relationships between tables remain consistent (foreign key must match a primary key in another table or be null).

Keys

  • Primary Key: Uniquely identifies each record in a table
  • Foreign Key: Establishes relationship between tables
  • Candidate Key: A field that could be a primary key
  • Alternate Key: A candidate key not chosen as primary key
  • Composite Key: A primary key made up of multiple columns

Normalization

Introduction

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. It involves dividing large tables into smaller, related tables and defining relationships between them.

Normal Forms

The stages of normalization are called normal forms:

1NF (First Normal Form)
  • Each table cell should contain a single value
  • Each record needs to be unique
  • No repeating groups or arrays
2NF (Second Normal Form)
  • Must be in 1NF
  • All non-key attributes must depend on the entire primary key (no partial dependency)
3NF (Third Normal Form)
  • Must be in 2NF
  • No transitive dependency (non-key attributes shouldn't depend on other non-key attributes)

Advantages

  • Reduced data redundancy
  • Improved data integrity
  • Better database organization
  • Easier database maintenance

Disadvantages

  • Increased complexity
  • More tables require more joins in queries
  • Potential performance impact

Centralized and Distributed Database

Introduction

Centralized Database: All data is stored at a single location and accessed by multiple users from that location.

Distributed Database: Data is stored across multiple locations (nodes) that are connected via network.

Advantages of Centralized Database

  • Easier to manage and maintain
  • Lower hardware and software costs
  • Better data consistency
  • Easier backup and recovery

Disadvantages of Centralized Database

  • Single point of failure
  • Performance bottleneck with many users
  • Network dependency for remote users

Advantages of Distributed Database

  • Improved performance (data closer to users)
  • Increased reliability (no single point of failure)
  • Scalability (easier to add more nodes)
  • Local autonomy (local control over local data)

Disadvantages of Distributed Database

  • More complex to design and implement
  • Higher hardware and software costs
  • More difficult to maintain data consistency
  • Security challenges

Database Security

Introduction

Database security refers to the protection of database from unauthorized access, misuse, or destruction. It includes various measures to ensure confidentiality, integrity, and availability of data.

Challenges

  • Unauthorized access
  • Data corruption
  • Data theft
  • SQL injection attacks
  • Denial of service attacks

Security Measures

  • Authentication: Verifying user identity
  • Authorization: Granting appropriate access rights
  • Encryption: Protecting sensitive data
  • Auditing: Tracking database activities
  • Backup and Recovery: Protecting against data loss

Roles of DBA (Database Administrator)

  • Installing and upgrading database software
  • Designing database structure
  • Implementing security measures
  • Managing user accounts and permissions
  • Monitoring database performance
  • Planning backup and recovery strategies
  • Troubleshooting database issues

Practical Topics: DDL and DML Languages

DDL (Data Definition Language): Used to define and modify database structure (CREATE, ALTER, DROP, TRUNCATE, RENAME).

DML (Data Manipulation Language): Used to manipulate data within database objects (SELECT, INSERT, UPDATE, DELETE).

3. Explain the different models of DBMS with advantages and disadvantages.

1. Hierarchical Model:

Tree-like structure with parent-child relationships. Each parent can have many children, but each child has only one parent.

Advantages: Simple structure, fast access for hierarchical data

Disadvantages: Limited to one-to-many relationships, rigid structure

2. Network Model:

Extension of hierarchical model that allows many-to-many relationships.

Advantages: More flexible than hierarchical, handles complex relationships

Disadvantages: Complex to design and implement

3. Relational Model:

Data organized in tables with rows and columns. Relationships established through foreign keys.

Advantages: Simple structure, flexible queries, data independence

Disadvantages: Performance issues with complex queries

4. Entity-Relational Model:

Conceptual model describing data as entities, attributes, and relationships.

Advantages: Good for database design, easy to understand

Disadvantages: Not directly implementable, needs conversion to relational model

4. What is a relational database? How is it different from other database models?

Relational Database: A database that organizes data into tables (relations) with rows and columns, where relationships between tables are established through foreign keys.

Differences from other models:

  • Unlike hierarchical/network models, it doesn't use physical pointers but logical relationships
  • More flexible than hierarchical model which only allows one-to-many relationships
  • Simpler structure than network model which can become complex with many relationships
  • Uses SQL for querying unlike other models which use specialized languages

5. What is data redundancy? How does DBMS help in reducing it?

Data Redundancy: Duplication of data in multiple locations in a database, which can lead to inconsistency and waste storage space.

How DBMS reduces redundancy:

  • Normalization process divides data into related tables
  • Foreign keys establish relationships instead of duplicating data
  • Centralized data storage eliminates multiple copies
  • Data integrity constraints prevent inconsistent duplicates

8. Define normalization. Explain 1NF, 2NF and 3NF with suitable examples.

Normalization: The process of organizing data in a database to reduce redundancy and improve data integrity by dividing large tables into smaller, related tables.

1NF (First Normal Form):

  • Each table cell contains a single value
  • Each record is unique
  • No repeating groups or arrays
  • Example: A student table with separate rows for each subject rather than a list of subjects in one cell

2NF (Second Normal Form):

  • Must be in 1NF
  • All non-key attributes depend on the entire primary key (no partial dependency)
  • Example: If a composite primary key (student_id + course_id) determines grade, then student_name shouldn't be in this table as it only depends on student_id

3NF (Third Normal Form):

  • Must be in 2NF
  • No transitive dependency (non-key attributes shouldn't depend on other non-key attributes)
  • Example: In a student table, if student_id determines department, and department determines HOD, then HOD should be moved to a department table

9. Explain the terms: primary key, foreign key, and candidate key with examples.

Primary Key: A column or set of columns that uniquely identifies each row in a table. Example: student_id in a Students table.

Foreign Key: A column that creates a relationship between two tables by referencing the primary key of another table. Example: department_id in Students table that references department_id in Departments table.

Candidate Key: A column or set of columns that could serve as a primary key (uniquely identifies rows and doesn't contain nulls). Example: In a Students table, both student_id and email could be candidate keys if both are unique.

10. What is SQL? Explain its components and common functions.

SQL (Structured Query Language): A standard language for managing and manipulating relational databases.

Components:

  • DDL (Data Definition Language): CREATE, ALTER, DROP (define database structure)
  • DML (Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE (manipulate data)
  • DCL (Data Control Language): GRANT, REVOKE (control access)
  • TCL (Transaction Control Language): COMMIT, ROLLBACK (manage transactions)

Common Functions:

  • Aggregate: COUNT(), SUM(), AVG(), MAX(), MIN()
  • String: CONCAT(), SUBSTRING(), UPPER(), LOWER()
  • Date: NOW(), DATE(), YEAR(), MONTH()
  • Mathematical: ROUND(), ABS(), POWER()

12. Define the following terms.

a. Data Dictionary: A centralized repository of metadata (data about data) that describes the structure, relationships, and other characteristics of the data stored in a database.

b. Primary Key: A column or set of columns that uniquely identifies each row in a table. It cannot contain NULL values and must be unique.

c. Relationship: An association between two or more tables in a database, established through common fields (typically primary and foreign keys).

d. Data Manipulation Language (DML): A subset of SQL used to retrieve, insert, update, and delete data in a database (e.g., SELECT, INSERT, UPDATE, DELETE).

e. Structured Query Language (SQL): A standard programming language used to manage and manipulate relational databases.

f. Data Integrity: The accuracy, consistency, and reliability of data stored in a database, maintained through various constraints and rules.

g. Data Definition Language (DDL): A subset of SQL used to define and modify database structures (e.g., CREATE, ALTER, DROP).

h. Data Security: Protective measures implemented to prevent unauthorized access, modification, or destruction of database information.

i. Database System: A complete system for database management including the database, DBMS, and application programs that interact with the database.

Chapter 2: C Programming II

1. Working with a function

a. Define function: A function is a self-contained block of code that performs a specific task. It can take inputs, process them, and return a result.

b. Syntax of functions:

return_type function_name(parameters) {
    // function body
    return value;
}

c. Types of functions:

  • Library functions: Predefined functions in C standard library (e.g., printf(), scanf())
  • User-defined functions: Functions created by programmers to perform specific tasks

d. Components of function:

  1. Function prototype: Declaration of function before its use (return type, name, parameters)
  2. Function call: Invoking the function to execute its code
  3. Function definition: Actual implementation of the function
  4. Return type: Data type of value the function returns (void if no return)

2. Categories of function with example

i. Function with return type but no arguments:

int getNumber() {
    int num;
    printf("Enter a number: ");
    scanf("%d", &num);
    return num;
}

ii. Function with return type with arguments:

int add(int a, int b) {
    return a + b;
}

iii. Function with no return type with no arguments:

void greet() {
    printf("Hello, World!");
}

iv. Function with no return type with arguments:

void printSum(int a, int b) {
    printf("Sum: %d", a + b);
}

3. Storage classes

i. Automatic (auto): Default storage class for local variables (created when function is called, destroyed when function exits)

void func() {
    auto int x;  // same as: int x;
}

ii. External (extern): Used to declare global variables visible to all files in a program

// file1.c
int count;  // global variable

// file2.c
extern int count;  // access global count from file1

iii. Register: Suggests compiler to store variable in CPU register for faster access

register int counter;

iv. Static: Preserves variable value between function calls, or limits scope to file

void counter() {
    static int count = 0;  // retains value between calls
    count++;
    printf("%d", count);
}

4. Recursive function with syntax and examples

Recursive function: A function that calls itself directly or indirectly to solve a problem by breaking it into smaller sub-problems.

Syntax:

return_type function(parameters) {
    if (base_condition)  // stopping condition
        return value;
    else
        return function(modified_parameters);  // recursive call
}

Example (Factorial):

int factorial(int n) {
    if (n == 0 || n == 1)  // base case
        return 1;
    else
        return n * factorial(n - 1);  // recursive call
}

Example (Fibonacci):

int fibonacci(int n) {
    if (n <= 1)  // base case
        return n;
    else
        return fibonacci(n-1) + fibonacci(n-2);  // recursive call
}

5. Demonstration of passing array to a function (code examples)

Passing array to function: Arrays can be passed to functions by specifying the array name without brackets.

Example 1: Passing 1D array

void printArray(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);
    }
}

int main() {
    int numbers[5] = {1, 2, 3, 4, 5};
    printArray(numbers, 5);  // pass array name and size
    return 0;
}

Example 2: Passing 2D array

void printMatrix(int mat[][3], int rows) {
    for (int i = 0; i < rows; i++) {
        for (int j = 0; j < 3; j++) {
            printf("%d ", mat[i][j]);
        }
        printf("\n");
    }
}

int main() {
    int matrix[2][3] = {{1, 2, 3}, {4, 5, 6}};
    printMatrix(matrix, 2);  // pass 2D array
    return 0;
}

Example 3: Modifying array in function

void doubleElements(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        arr[i] *= 2;  // original array will be modified
    }
}

int main() {
    int nums[3] = {5, 10, 15};
    doubleElements(nums, 3);
    // nums now contains {10, 20, 30}
    return 0;
}
